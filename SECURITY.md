# Security Policy

## Reporting a vulnerability
Please do not open public issues for security vulnerabilities.

Use private disclosure to maintainers and include:
- Affected component/file
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

## Response targets
- Initial acknowledgment: within 3 business days
- Triage update: within 7 business days
- Fix timeline: based on severity and blast radius

## Secrets and PII
- Never commit API keys or production secrets.
- Redact user data/PII in logs, examples, and bug reports.
